Frequently Asked Questions

General Questions

What is steganography? ‚ñº
Steganography is the art of hiding secret messages inside ordinary files like images, audio, or video. Unlike encryption which scrambles data, steganography conceals the existence of the message itself, making it invisible to casual observers.
How does this tool work? ‚ñº
Our tool uses the LSB (Least Significant Bit) technique. It modifies the least significant bits of pixel RGB values to hide your message. These tiny changes are invisible to the human eye but can be extracted later to reveal the hidden text.
Is my data safe? ‚ñº
Yes! All processing happens entirely in your browser using JavaScript. No images or messages are ever sent to any server. Your data never leaves your device, ensuring complete privacy and security.

Technical Questions

What image formats are supported? ‚ñº
Only PNG format is supported. PNG is a lossless format, meaning pixel data remains unchanged after saving. This is crucial for steganography as any compression would destroy the hidden message.
What is the maximum message length? ‚ñº
The maximum message length is currently set to 500 characters. However, this also depends on your image size. A larger image can theoretically hold more data. Generally, a 1920√ó1080 image can comfortably hide 500-800 characters.
Can I use JPEG images? ‚ñº
No, JPEG uses lossy compression which modifies pixel values during the save process. This would destroy any hidden messages. You must always use PNG format for both encoding and decoding.
How do I decode a message? ‚ñº
Go to the Decode page, upload the encoded PNG image (the one you downloaded after encoding), and click "Extract Message". The hidden text will appear in the output box. You can then copy it to your clipboard.

Troubleshooting

Why can't I extract the message? ‚ñº
Make sure you're using the exact PNG image that was downloaded after encoding. If the image was converted to JPEG, edited, or compressed, the hidden message will be lost. Also ensure you're using the same tool for both encoding and decoding.
The image is too large. What should I do? ‚ñº
Try using a smaller image or resize your image before encoding. For web use, images around 1920√ó1080 pixels work well. Remember to keep it in PNG format when resizing.
My browser shows an error ‚ñº
Make sure you're using a modern browser (Chrome, Firefox, Edge, or Safari). Clear your browser cache and try again. If the problem persists, try using a different browser.
